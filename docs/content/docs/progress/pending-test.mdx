# 待测试

- 用户体系：注册、登录、登出、会话恢复、未登录拦截是否完整可用。
- 超级管理员：是否只能由 `admin` 看到并编辑共享配置、用户管理、模型点数配置。
- 共享配置：渠道、模型、模型点数、WebDAV 配置保存后，普通用户是否只能读取不能修改。
- 点数系统：
  - 管理员调整余额/额度是否生效。
  - 图片生成、图片编辑、视频生成、音频生成、文本生成/工具调用是否按模型点数扣减。
  - 失败场景下点数是否自动退回。
- Postgres 持久化：
  - 画布、素材、图片/视频生成记录是否按用户隔离保存。
  - 文件上传后是否写入 `user_files`。
- 全站资源管理：
  - Base64、外部 URL 和本地上传产生的图片、视频、音频是否同时写入 `user_files` 与 `user_resources`；画布生成文本、助手最终回复和素材文本是否登记为文本资源。
  - 管理员资源页的关键词、用户、类型、收藏状态、来源、日期筛选，以及 20/50/100 服务端分页、分类数量和存储占用是否正确。
  - 图片预览、视频/音频 Range 播放、文本查看和原文件下载是否正常，普通用户是否无法访问页面和 API。
  - 删除节点、连线、画布、生成记录、助手会话或素材条目后资源是否仍保留；加入/移出素材时 `is_saved` 是否同步。
  - 管理员单项、批量和清空全部是否能删除已收藏资源；引用节点和素材卡片是否显示“资源已删除”，文本节点是否保留内容副本。
  - 四类自动保留天数是否以 `0` 为关闭、按 `created_at` 清理并跳过已收藏资源；立即清理、每日任务和多实例 advisory lock 是否符合预期。
  - 旧文件、文本素材和可识别的生成文本是否只回填一次，旧媒体时间是否使用 `user_files.updated_at`，且不重复复制文件正文。
- 首次登录迁移：旧浏览器本地画布、素材、生成记录是否能自动迁入 Postgres。
- 图片异步生成兼容：
  - OpenAI 普通生图、图片编辑和 Gemini `generateContent` 是否默认携带 `async=true`，同步图片响应是否仍可直接使用。
  - `task/task_id/taskId/id` 及常见任务状态是否能正确解析，任务成功后的 OpenAI Base64/URL，以及顶层或 `response` 内的 Gemini candidates 图片是否正常展示和保存。
  - 不支持异步的接口是否只同步回退一次并在当前会话缓存；提示词或尺寸错误是否不会误记为不支持异步。
  - 任务创建后的临时 `404`、限流、网络错误和 `5xx` 是否只重试查询，不重复创建任务；失败、取消、过期和超时是否正确结束。
- 画布加载与保存优化：
  - 多个画布相关请求首次并发访问空数据库时，用户数据表只初始化一次且请求均可正常完成。
  - 画布列表只返回摘要，进入页面只读取当前项目详情。
  - 当前画布在媒体下载前即可显示，视口外图片、视频和聊天附件不会提前加载。
  - 重新进入画布时图片节点使用最长边 1280px 的 WebP 预览缓存，新图片上传时生成、旧图片首次访问自动补齐；生成完成的当前标签页仍直接显示内存原图，下载和 AI 编辑继续读取原文件。
  - 图片使用稳定鉴权文件 URL，视频/音频使用 metadata 预加载，视频 Range 请求返回 `206` 并按分片读取 PostgreSQL `BYTEA`。
  - 连续编辑只保存当前画布，离开页面、退出和切换账号前会刷新最后一笔防抖保存。
  - 创建、重命名、批量删除、导入导出、WebDAV 同步和资源引用保留在新单项目接口下是否完整可用。
- 生成配置提示词组装：连续点击生成时，上游文字只动态解析一次，`@[node:id]` 模板保持不变；修改源文字后下一次生成使用最新内容且不累积重复。
- 提示词库封面缓存：封面是否能正常命中缓存，清理策略是否按预期执行。
- 顶部操作区：GitHub / 版本入口已移除，“关于”按钮弹窗展示是否正确。

- 多 Base URL 锁定：
  - LOCKED_CHANNEL_BASE_URLS 环境变量支持逗号分隔多个中转地址。
  - 普通用户渠道的 Base URL 改为下拉选择器（Select），只能从管理员设置的列表中选取。
  - 修复普通用户初始化时锁定 URL 列表未写入前端状态、下拉无选项，以及 Base URL 下方提示乱码的问题。
  - Docker Compose 显式向应用容器传递单数和复数锁定 URL 环境变量。
  - AI 模型列表、生成、查询和流式请求统一经过 `/api/ai-proxy` 同源转发，不再要求上游接口支持浏览器 CORS。
  - AI 代理在请求阶段校验 Base URL：普通用户只能访问 `LOCKED_CHANNEL_BASE_URLS` 范围，超管可访问任意 HTTP/HTTPS URL，重定向也执行同样校验。
  - 超管不接收锁定列表，读取和保存配置时均不执行 `LOCKED_CHANNEL_BASE_URLS` 白名单校验，可以自由使用任意 Base URL。
  - 已有的单 LOCKED_CHANNEL_BASE_URL 变量向后兼容，自动转为单元素列表。
  - 服务端 enforceLockedBaseUrl 白名单校验：用户保存的 baseUrl 不在列表中时自动回退到第一个。
- 全站 Agent：新增 `generation_get_status` 工具，画布生成节点可按 `nodeIds` 查询，生图和视频工作台提交后会返回 `taskId` 并可查询排队、运行、成功或失败状态；需验证查询只由当前活动标签页返回。
- 本地 Agent 多标签页隔离：同时打开两个不同画布并连接同一个 Agent，分别聚焦标签页后通过 MCP 读取和修改画布，操作应只落在当前聚焦页面；网页面板发起的整个 Codex turn 应固定操作发起页面，即使中途聚焦另一标签页也不能切换目标；关闭当前页面后应回退到最近聚焦且仍连接的页面，其他页面回传同一请求结果应被拒绝。
- 本地 Agent 多标签页会话同步：所有标签页共享同一个站点级 Codex 活跃线程；任一页面发送消息、新建、恢复或删除会话后，其他页面应同步活跃线程和聊天记录；Agent 输出仅显示在事件所属线程，运行中不能新建、恢复、删除或再次发送任务。
- 本地 Agent 运行状态同步：在一个标签页运行较长 Codex 任务，等待某张工具卡显示「工具完成」后再打开或刷新第二个标签页；第二个标签页应立即显示 Codex 正在运行并禁用发送，整轮结束后两个标签页同时恢复；工具卡只显示「工具完成」，整轮结束由「本轮完成」表示。
- 本地 Agent 图片附件落画布：在右侧 Agent 上传参考图并要求基于商品信息创建生图流程，附件应创建为保持原比例的真实图片节点，分析提示词应创建为文本节点，二者都应连接到生成配置节点；刷新页面后参考图仍可显示并参与生成。任务中途切换到其他标签页时，附件只能写入发起任务的标签页；若发起标签页关闭，附件读取应失败且不能落入其他画布。
- Agent 对话滚动：从历史或日志切回对话、恢复其他会话时应自动定位到最新消息；手动向上浏览后应显示向下箭头且新消息不强制打断阅读，点击箭头后平滑回到底部并继续跟随新消息。
- Agent 消息区分：用户消息应在右侧以主题浅底气泡显示，AI 回复应在左侧保留 Agent 图标并使用开放式 Markdown 排版；长文本、多张附件、错误消息及浅色/深色主题下均应清晰且不溢出。
