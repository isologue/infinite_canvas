# 待测试

- 用户体系：注册、登录、登出、会话恢复、未登录拦截是否完整可用。
- 超级管理员：是否只能由 `admin` 看到并编辑共享配置、用户管理、模型点数配置。
- 共享配置：渠道、模型、模型点数、WebDAV 配置保存后，普通用户是否只能读取不能修改。
- 点数系统：
  - 管理员调整余额/额度是否生效。
  - 图片生成、图片编辑、视频生成、音频生成、文本生成/工具调用是否按模型点数扣减。
  - 失败场景下点数是否自动退回。
- Postgres 持久化：
  - 画布、素材、图片/视频生成记录是否按用户隔离保存。
  - 文件上传后是否写入 `user_files`。
- 首次登录迁移：旧浏览器本地画布、素材、生成记录是否能自动迁入 Postgres。
- 提示词库封面缓存：封面是否能正常命中缓存，清理策略是否按预期执行。
- 顶部操作区：GitHub / 版本入口已移除，“关于”按钮弹窗展示是否正确。

- 多 Base URL 锁定：
  - LOCKED_CHANNEL_BASE_URLS 环境变量支持逗号分隔多个中转地址。
  - 普通用户渠道的 Base URL 改为下拉选择器（Select），只能从管理员设置的列表中选取。
  - 超管不接收锁定列表，读取和保存配置时均不执行 `LOCKED_CHANNEL_BASE_URLS` 白名单校验，可以自由使用任意 Base URL。
  - 已有的单 LOCKED_CHANNEL_BASE_URL 变量向后兼容，自动转为单元素列表。
  - 服务端 enforceLockedBaseUrl 白名单校验：用户保存的 baseUrl 不在列表中时自动回退到第一个。
